<objective>
Implement three tempo stability improvements in OnsetDetector to reduce BPM jitter when
playing through a fuzz pedal: BPM lock-in after consistent IOIs, BPM quantization to the
nearest 5-BPM grid, and a wider onset refractory window to reject harmonic false-triggers.

Purpose: Fuzz guitar generates rich harmonics that trigger multiple spurious onsets per
note. These three changes together make the tempo estimate snap to stable integer-grid values
and stop chasing noise once real tempo is established.

Output: Modified OnsetDetector.h/.cpp and one call-site change in AccompanimentProcessor.cpp.
</objective>

**Fix 3 — BPM lock-in (runFftFrame):**
Inside the `if (ioi >= minIoi && ioi <= maxIoi)` block, after `pushIoi(ioi)` and before or replacing the unconditional `currentBpm.store(...)`, add the lock-in logic:

```cpp
pushIoi(ioi);

if (!tempoLocked)
{
    const float bpm = medianIoiBpm();

    // Lock-in check: are all accumulated IOIs within ±8% of their mean?
    if (ioiRingCount >= 8)
    {
        float sum = 0.0f;
        for (int i = 0; i < ioiRingCount; ++i)
            sum += ioiHistory[static_cast<size_t>(i)];
        const float mean = sum / static_cast<float>(ioiRingCount);
        const float threshold = mean * 0.08f;

        bool allConsistent = true;
        for (int i = 0; i < ioiRingCount; ++i)
        {
            if (std::abs(ioiHistory[static_cast<size_t>(i)] - mean) > threshold)
            {
                allConsistent = false;
                break;
            }
        }
        if (allConsistent)
            tempoLocked = true;
    }

    currentBpm.store(bpm, std::memory_order_relaxed);
}
```
