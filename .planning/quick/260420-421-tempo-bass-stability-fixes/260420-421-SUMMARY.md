## What Was Done

### Task 1 — Octave fold in `medianIoiBpm()`

In `OnsetDetector::medianIoiBpm()`, after computing `bpm = 60.0f / ioiSec`, added octave disambiguation before the `jlimit` clamp. If the computed BPM is more than 1.7x the current estimate it is halved; if less than 0.6x it is doubled. The guard `current > 0.0f` prevents the fold from running on the very first estimate when `currentBpm` is still the 120 BPM default.

This prevents the detector locking onto 8th-note IOIs (2x BPM) or half-note IOIs (0.5x BPM) when a guitarist plays at a steady tempo.

### Task 2 — Exponential smoothing in `PatternPlayer::setBpm()`

Replaced the hard assignment `bpm = juce::jlimit(...)` with `bpm += 0.1f * (clamped - bpm)`. Alpha = 0.1 gives a time constant of approximately 10 calls (~200 ms at 50 Hz) — fast enough to track real tempo changes while preventing beat-clock stutter from individual noisy BPM estimates.

### Task 3 — Pitch stability gate + 2-bar bass root hold

**AccompanimentProcessor.h:** Added three plain members:
- `int pitchStableCounterSamples = 0`
- `float lastStablePitchMidi = 40.0f`
- `int64_t lastBassUpdateSample = -1`

**processBlock() pitch stability gate:** The existing single-block semitone commit is replaced with a two-phase gate. The counter accumulates samples while pitch stays within ±0.5 semitones of the reference. When pitch jumps, the counter resets and the reference updates. `setBassSemitoneOffset` is only called once the counter exceeds one beat duration in samples at the current BPM. This suppresses transient pitch noise without adding perceptible latency at normal tempos.

**prepareToPlay() resets:** The three new members are reset alongside the existing `heldPitchRootMidi`, `heldPitchConfidence`, `pitchHoldValid` resets.

**drainFeatureQueueAndRunInference() 2-bar hold guard:** Added `bassHoldExpired` computed from `lastBassUpdateSample` and a two-bar window (8 beats at current BPM). Both the `generativeMode == 1` branch and the auto-mode `else` branch now only store a new `generativeBassRootNote` + `generativeBassDurationBeats` when the hold has expired. During the hold window the existing root note is preserved; `useGenerativeBass` is still set to true so playback continues uninterrupted.
