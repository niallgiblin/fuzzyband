**4-onset count-in gate delays drums until rhythm is established, 2-bar hold prevents pattern flickering, both using sample-timestamp comparisons consistent with existing bass hold pattern**

## Decisions Made
- `snapToBarStart()` deliberately does not call `reset()` to preserve sampleCounter and generative bass note-off state — only the beat clock and pending pattern need clearing
- Plain (non-atomic) types for count-in state: all reads/writes are audio-thread-only, no cross-thread access
- `lastDrumPatternChangeSample` uses the same plain-int64_t pattern as `lastBassUpdateSample` — the inference thread writes it and the audio thread resets it only during SILENT (when no MIDI output is possible), making the race harmless
