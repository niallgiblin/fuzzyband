Session round-trip test contract (from 24-UI-SPEC.md):
  Location: tests/test_processor_pipeline.cpp
  Test name: "Session XML round-trip: pre-v0.4.0 genre attribute ignored"
  Fixture: Construct minimal APVTS XML string containing a genre attribute plus current params
  Assertion 1: setStateInformation does not throw or crash (REQUIRE_NOTHROW or CHECK)
  Assertion 2: intensity parameter loads to the fixture value within 0.01f tolerance
  Assertion 3: Plugin remains in consistent state after load (pattern index in [0,6], no null deref)

How JUCE APVTS serializes state (relevant to constructing the XML fixture):
  - JUCE APVTS.getStateInformation() produces XML with element name matching the processorID
  - Parameters are stored as attributes on a child element named "PARAM" with "id" and "value"
  - A pre-v0.4.0 session XML would have been produced by the same JUCE APVTS mechanism
    but with genre and variation present. JUCE ignores unknown PARAM elements on load.
  - Construct the XML manually as a string, convert to MemoryBlock, then call setStateInformation.

Minimal XML fixture approach:
  Build a juce::XmlElement tree programmatically (safer than raw string) or use a raw XML string.
  Recommended (raw string, simpler):
  ```
  const juce::String xmlStr =
      R"(<APVTS>"
      R"(  <PARAM id="genre" value="0.75"/>)"
      R"(  <PARAM id="intensity" value="0.6"/>)"
      R"(  <PARAM id="structureBlend" value="0.4"/>)"
      R"(  <PARAM id="variation" value="0.2"/>)"
      R"(</APVTS>)";
  ```
  But note: the actual JUCE APVTS XML root element name is the processor's name or a default.
  Safer approach: call getStateInformation on a fresh processor first to discover the XML root
  element name, then construct a matching XML with unknown attributes added.

  Simplest reliable approach for the fixture:
  1. Instantiate AccompanimentProcessor
  2. Set intensity to 0.6f and structureBlend to 0.3f
  3. Call getStateInformation → data block (this is a valid v0.4.0 XML)
  4. Parse data block to juce::XmlElement
  5. Add a fake "genre" child PARAM element (id="genre", value="0.75") and a "variation" child (id="variation", value="0.2")
  6. Serialise back to MemoryBlock
  7. Call setStateInformation on a second fresh processor
  8. Assert: intensity ≈ 0.6f, structureBlend ≈ 0.3f, patternIndex in [0,6]

```cpp
// ─── Session backward-compat: pre-v0.4.0 XML with genre attribute ─────────────

TEST_CASE("Session XML round-trip: pre-v0.4.0 genre attribute ignored", "[integration][pipeline]")
{
    // Step 1: Build a baseline v0.4.0 state from a fresh processor with known param values.
    AccompanimentProcessor procSrc;
    procSrc.prepareToPlay(48000.0, 512);

    if (auto* p = procSrc.getApvts().getParameter("intensity"))
        p->setValue(0.6f);
    if (auto* p = procSrc.getApvts().getParameter("structureBlend"))
        p->setValue(0.3f);

    juce::MemoryBlock baseData;
    procSrc.getStateInformation(baseData);
    REQUIRE(baseData.getSize() > 0);

    // Step 2: Parse the serialized state XML and inject pre-v0.4.0 unknown params.
    auto xmlDoc = juce::XmlDocument::parse(juce::String::fromUTF8(
        static_cast<const char*>(baseData.getData()),
        static_cast<int>(baseData.getSize())));
    REQUIRE(xmlDoc != nullptr);

    // Inject pre-v0.4.0 attributes that no longer exist in the APVTS.
    // JUCE silently ignores unknown PARAM elements on deserialization.
    auto* genreParam = xmlDoc->createNewChildElement("PARAM");
    genreParam->setAttribute("id", "genre");
    genreParam->setAttribute("value", "0.75");

    auto* varParam = xmlDoc->createNewChildElement("PARAM");
    varParam->setAttribute("id", "variation");
    varParam->setAttribute("value", "0.2");

    // Step 3: Serialise the tampered XML back to a MemoryBlock.
    juce::MemoryOutputStream tamperedStream;
    xmlDoc->writeTo(tamperedStream);
    const juce::String tamperedXml = tamperedStream.toString();
    const auto tamperedBytes = tamperedXml.toRawUTF8();
    const size_t tamperedSize = tamperedXml.getNumBytesAsUTF8();
    juce::MemoryBlock tamperedData(tamperedBytes, tamperedSize);

    // Step 4: Load tampered state into a fresh processor — must not crash.
    AccompanimentProcessor procDst;
    procDst.prepareToPlay(48000.0, 512);
    procDst.setStateInformation(tamperedData.getData(), static_cast<int>(tamperedData.getSize()));

    // Step 5: Surviving parameters must load correctly.
    if (auto* inten = dynamic_cast<juce::AudioParameterFloat*>(
            procDst.getApvts().getParameter("intensity")))
        REQUIRE(std::abs(inten->get() - 0.6f) < 0.01f);

    if (auto* blend = dynamic_cast<juce::AudioParameterFloat*>(
            procDst.getApvts().getParameter("structureBlend")))
        REQUIRE(std::abs(blend->get() - 0.3f) < 0.01f);

    // Step 6: Plugin must be in a consistent state (pattern index in valid range).
    REQUIRE(procDst.getDisplayPatternIndex() >= 0);
    REQUIRE(procDst.getDisplayPatternIndex() <= 6);

    procSrc.releaseResources();
    procDst.releaseResources();
}
```

Note on XmlDocument: JUCE's `XmlDocument::parse()` signature returns `std::unique_ptr<juce::XmlElement>`. Adjust the null check and pointer usage accordingly if `xmlDoc` is a unique_ptr — use `xmlDoc.get()` for raw pointer calls and `REQUIRE(xmlDoc != nullptr)` for the null guard.
