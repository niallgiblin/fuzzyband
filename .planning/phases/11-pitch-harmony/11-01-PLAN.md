<task type="execute">
  <name>Task 3: Wire PitchEstimator + D-11-04 policy in AccompanimentProcessor</name>
  <files>src/AccompanimentProcessor.h, src/AccompanimentProcessor.cpp</files>
  <read_first>

    - .planning/phases/11-pitch-harmony/11-CONTEXT.md (D-11-04 exact behavior)
  </read_first>
  <action>
    1. Add a private member `PitchEstimator pitchEstimator;` and optional small state for **last held** `pitchRootMidi` / `pitchConfidence` (floats) in `AccompanimentProcessor` — only if needed for hold-last; may be locals reset in `prepareToPlay` via `reset()`.

    2. In `prepareToPlay`, call `pitchEstimator.prepare(sampleRate, samplesPerBlock)` and `pitchEstimator.reset()`.

    3. In `processBlock`, after existing analysis and **before** building `FeatureVector fv`:

       - Call `pitchEstimator.process(in, numSamples)` on the mono guitar buffer `in`.
       - Read raw `midi = pitchEstimator.getMidiNote()`, `conf = pitchEstimator.getConfidence()`.
       - **Silence / cold start:** If `st == StructureState::SILENT` OR `rms` is below `1.0e-6f` (document as “digital silence floor”), set effective root to **40.0f**, confidence **0.0f**, and clear hold state as appropriate so bass policy snaps to E2.
       - **Else if** `conf` below a named constant `kMinPitchConfidence` (e.g. `0.35f` defined once in `.cpp`), **retain** previous held root and confidence (clamp confidence down if needed) — do not jump to 40 unless silence rule fired.
       - **Else** update held root to `midi` and confidence to `conf`.

    4. Assign `fv.pitchRootMidi` and `fv.pitchConfidence` to the **effective** values after policy.

    5. Do not change `inferenceLoop` or `IInference` contracts in this plan (Phase 11 plan 02 may consume pitch for bass only).

    6. Optionally mirror pitch to new `std::atomic<float>` display members — only if trivial; otherwise skip.
  </action>
  <acceptance_criteria>

    - `grep -n "fv.pitchRootMidi\\|fv.pitchConfidence" src/AccompanimentProcessor.cpp` exits 0.
