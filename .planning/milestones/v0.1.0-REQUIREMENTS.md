## v1 Requirements (Phase 1 — Rule-Based MVP)

### Scaffold & Build (SCAF)

- [x] **SCAF-01**: Plugin builds as VST3 + AU targets via CMake on macOS
- [x] **SCAF-02**: Plugin loads in a DAW (Reaper), shows blank UI, processes silence without crashing
- [x] **SCAF-03**: AudioProcessorValueTreeState skeleton in place for future parameter UI
- [x] **SCAF-04**: `.gitignore` covers JUCE/CMake artifacts
- [x] **SCAF-05**: GitHub Actions CI builds on macOS (build-only runner)

### Onset & Tempo (ONSET)

- [x] **ONSET-01**: Spectral flux onset detector using JUCE `dsp::FFT`; outputs onset timestamps as sample positions
- [x] **ONSET-02**: IOI-based tempo tracker; median filter over last 8 IOIs; clamped to 80–220 BPM
- [x] **ONSET-03**: Thread-safe `getCurrentBpm()` and per-block onset flag
- [x] **ONSET-04**: Unit test: synthetic click track at 160 BPM → tracker converges within 4 beats

### Energy & Structure (ENERGY)

- [x] **ENERGY-01**: EnergyAnalyser: RMS over 100ms rolling window, spectral centroid, low-freq flux
- [x] **ENERGY-02**: StructureTagger: SILENT / VERSE / CHORUS / BREAKDOWN states with 2s hysteresis
- [x] **ENERGY-03**: Debug UI overlay showing current state label and BPM

### MIDI Patterns (MIDI)

- [x] **MIDI-01**: MidiPattern struct: `{note, velocity, beatOffset, duration}`; 7 addressable pattern indices
- [x] **MIDI-02**: Minimum 3 drum patterns per structural state (VERSE, CHORUS, BREAKDOWN)
- [x] **MIDI-03**: Corresponding bass patterns (root whole notes / root+fifth / syncopated root)
- [x] **MIDI-04**: Patterns stored as C++ `constexpr` arrays (no file I/O in plugin)
- [x] **MIDI-05**: Offline MIDI export utility to audition patterns in a DAW

### MIDI Output (OUT)

- [x] **OUT-01**: PatternPlayer: beat clock derived from BPM + sample position; fills MidiBuffer
- [x] **OUT-02**: Pattern looping and smooth transitions between patterns
- [x] **OUT-03**: Humanisation: velocity ±10, timing ±2ms randomisation
- [x] **OUT-04**: MIDI channel 10 (drums) and channel 2 (bass) separated
- [x] **OUT-05**: Reaper session verified — drum VSTi triggered at correct rhythmic positions *(code + routing complete; optional maintainer DAW smoke-test remains)*

### Inference (INFER)

- [x] **INFER-01**: `IInference` abstract interface: `virtual int selectPattern(const FeatureVector& f) = 0`
- [x] **INFER-02**: `RuleBasedInference`: SILENT → stop MIDI; VERSE/CHORUS/BREAKDOWN → BPM-band selection
- [x] **INFER-03**: Inference runs on background thread at ~50Hz
- [x] **INFER-04**: Lock-free pipeline: `ReaderWriterQueue` feature queue + `std::atomic<int>` pattern index

### Stability & Integration (STAB)

- [x] **STAB-01**: 10-minute jam session profiled; thresholds and hysteresis tuned by ear — *evidence: `07-05-JAM-NOTES.md`*
- [x] **STAB-02**: CPU usage < 15% on M-series Mac at 256-sample buffer — *evidence: `07-05-CPU-PROFILE.md`*
- [x] **STAB-03**: Graceful handling: plugin bypassed, sample rate change, buffer size change, transport stop/start — *evidence: `07-05-EDGE-CASE-REPORT.md`*
- [x] **STAB-04**: 20-minute session stable — no crashes, no xruns — *evidence: `07-05-STABILITY-REPORT.md`*

### Docs & Handoff (DOCS)

- [x] **DOCS-01**: ARCHITECTURE.md describing threading model and component boundaries
- [x] **DOCS-02**: Doxygen comments on all public class interfaces — *see Phase 8 verification grep in `08-01-SUMMARY.md`*
- [x] **DOCS-03**: `CONTRIBUTING.md` with build instructions for macOS (and Linux stretch goal)
- [x] **DOCS-04**: Tag `v0.1.0-phase1` release on GitHub — *annotated tag created locally; `git push origin v0.1.0-phase1` may still be pending*
- [x] **DOCS-05**: Open GitHub Issues for all known Phase 2 work items — *checklist maintained in `docs/PHASE2_GITHUB_ISSUES.md`; creating issues on GitHub is an operational follow-up*
