# Milestone v0.4.0: ML Playability & Simplification

**Status:** SHIPPED 2026-04-29  
**Phases:** 21–26  
**Total plans:** 13 (2 + 1 + 2 + 2 + 3 + 3)

## Overview

Make the plugin musical and self-contained — better-trained ML drives all pattern decisions from a richer dataset, the UI gets out of the way, and the structure model is simplified to energy states (SILENT / SOFT / LOUD). Training adds Lakh MIDI + GMD merge; ONNX contracts move to 3-class structure and `[1,32]` piano-roll bass; C++ inference enforces shapes and ML pattern rejection; human Reaper jam (PVAL-01) closes the milestone.

**Dependency note:** Phases 21 → 22 → 23 run in strict sequence. Phase 24 depends on Phase 23. Phase 25 depends on Phase 21 and can parallel 22–24. Phase 26 depends on Phase 23 and Phase 25.

**Requirements archive:** `.planning/milestones/v0.4.0-REQUIREMENTS.md`

## Milestone summary

**Key decisions**

- Three-class energy structure (SILENT/SOFT/LOUD) and removal of genre from APVTS before retraining — avoids silent ONNX mismatch.  
- Piano-roll bass `[1,32]` contract with load-time assertions; rule-based fallback only when model files absent.  
- ML “Next Pattern” via `patternRejectionCount` + `excludeIndex` instead of UI pattern cycling.  
- GMD + Lakh merge with domain compatibility and preserved GMD validation shard for comparison.  

**Technical debt / follow-ups**

- v0.5.0 rhythmic coherence (phases 27–30) — beat tracker, bar-aligned bass sequencer, enriched `FeatureVector`, fills, 12-feature retrain.
